# 任务
你将收到多集播客的结构化笔记。请将它们整合为一份综述报告，帮助读者快速了解这批播客涉及哪些领域、每集的核心内容是什么。目标篇幅约 4000 字。

# 输出格式要求
- 纯文本，不使用 Markdown 语法（不用 **加粗**、不用 `- 列表`、不用 `## 标题符号`、不用表格）
- 只允许使用中文数字编号标题（如"一、AI 与产品"）和段落正文
- 直接开始正文，不加任何前言或说明

# 核心原则
- 按领域分类，而非按观点聚合：将播客按所属领域（如 AI 与产品、产业与制造、内容创作、投资、文化等）归类，每个领域下列出相关播客
- 每集内容独立呈现：不强行将不同播客的观点放在一起对照，除非它们确实在讨论同一个具体议题
- 不生造关联：如果两集播客之间没有实质关联，不要为了结构完整而制造联系

# 输出结构

第一部分：本期速览

用一段约 150 字的文字，列出本期共整理多少集播客，按领域分布概述：每个领域包含几集、涉及什么话题。

第二部分：分领域详述

按领域设立章节（如"一、AI 与产品"、"二、产业与制造"），每个领域下按播客逐条展开。

每集播客的写法：

先写一行标题，格式为：【序号】播客主题或标题（来源：播客名称 / 嘉宾）

然后写核心内容段落：2—4 句话概括这集的中心论点或主要发现。

然后写关键细节段落：保留具体的案例、数据、时间节点、人名、公司名，用连续段落写出，每个细节点单独成句，不压缩。

然后写值得记录的原话：1—2 句有信息量的直接引语，标明是引语即可，不另起标题。

第三部分：本期播客清单

列出所有播客，格式为：序号、领域、播客名称、嘉宾、核心话题，每条一行。

# 领域分类参考（可根据实际内容调整）
- AI 与产品：大模型应用、AI 工具、Agent 架构、产品设计
- 产业与制造：供应链、硬科技、制造业、能源
- 投资与商业：VC 视角、公司战略、商业模式
- 内容创作：Newsletter、播客运营、写作方法论
- 文化与其他：音乐、体育、人文话题

# 写作风格
- 语言正式、准确，参考《经济学人》或《金融时报》的语气：冷静、精确，有信息量但不渲染情绪
- 每个细节点都要有具体内容，不用空泛概括代替事实
- 不使用"赋能"、"底层逻辑"、"范式转移"等空泛词汇
- 不为简洁牺牲信息量

# 禁止事项
- 不强行制造播客之间的关联
- 不将不同播客的观点混在一起表述
- 不在结尾进行抽象升华
- 不使用"震撼"、"颠覆"、"干货满满"等情绪化表达
- 不使用 Markdown 语法（包括 **、##、-、> 等符号）
