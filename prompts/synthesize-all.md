# 任务
你将收到多集播客的结构化笔记。请将它们整合为一份综述报告，帮助读者快速了解这批播客涉及哪些话题、每集的核心内容是什么。目标篇幅约 4000 字。

# 读者定位
读者对话题有基本兴趣，但不一定是行业内人士。综述要让门外汉也能读懂，遇到专业概念要顺带解释。

# 输出格式要求
- 纯文本，不使用 Markdown 语法（不用 **加粗**、不用 - 列表、不用 ## 标题符号）
- 只允许使用中文数字编号的章节标题（如"一、AI 与产品"）和正文段落
- 直接开始正文，不加任何前言

# 写作风格
参考《晚点》或《好奇心日报》的报道风格：
- 用具体的人、事、数字说话，不用抽象概括代替事实
- 句子短，逻辑清晰
- 保留嘉宾的原话，尤其是有判断力或有意思的表达
- 不使用"赋能"、"底层逻辑"、"叙事牵引"、"认知盲区"、"范式转移"等空泛词汇
- 遇到行业术语，第一次出现时解释一句

# 输出结构

第一部分：本期速览

用一段约 150 字的文字，说清楚本期整理了多少集播客、大致涉及哪些话题领域。读者看完这段就能决定要不要继续读。

第二部分：分领域详述

按话题领域分章节（如"一、AI 与产品"、"二、投资与商业"）。每个领域下，每集播客单独写，格式如下：

先写一行标题：【序号】播客主题（来源：播客名 / 嘉宾姓名）

然后写核心内容：用 2—4 句话说清楚这集主要讲了什么、嘉宾的核心判断是什么。

然后写关键内容：用连贯的段落展开，保留具体案例、数字、时间节点、人名、公司名。不要切成大量子弹点，要让人读起来顺畅。

然后写值得记住的话：引用 1—2 句嘉宾的原话。

第三部分：本期播客清单

每集一行，格式：序号、领域、播客名、嘉宾、核心话题。

# 领域分类参考（可根据实际内容调整）
- AI 与产品：大模型、AI 工具、Agent、产品设计
- 投资与商业：VC、公司战略、商业模式
- 产业与科技：供应链、硬件、制造业、能源
- 内容与创作：媒体、播客、写作
- 文化与其他：音乐、体育、人文

# 禁止事项
- 不强行制造播客之间的关联
- 不在结尾写抽象升华
- 不使用"震撼"、"颠覆"、"干货满满"、"赋能"、"底层逻辑"等词
- 不使用 Markdown 语法
